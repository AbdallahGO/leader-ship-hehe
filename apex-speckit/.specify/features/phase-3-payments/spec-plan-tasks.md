# Phase 3 — Spec: Stripe Payment Integration
**Spec Kit artifact** · Run `/speckit.specify` then `/speckit.clarify`

---

## Overview

Users can purchase Summit tickets through the payment modal.
Stripe handles all card processing. Our server creates PaymentIntents, confirms orders, and listens to webhook events.
On successful payment: order and tickets are saved to the database, QR code generated.

---

## Functional Requirements

### F1 — Create Payment Intent
`POST /api/orders/create` (protected — requires auth).
Accepts: `plan_key`, `quantity`, `promo_code` (optional).
Validates plan exists and is active. Applies promo discount if code valid.
Calculates: subtotal, fee (3%), total.
Creates Stripe `PaymentIntent` with `amount` (in cents), `currency: 'usd'`, metadata: `{ user_id, plan_key, quantity, promo_code }`.
Returns: `{ client_secret, order_preview: { plan, qty, subtotal, fee, total } }`.

### F2 — Frontend Payment Flow
Stripe.js loads on payment modal open with publishable key.
After Step 1 (personal info), Step 2 calls `POST /orders/create` to get `client_secret`.
Stripe Elements renders the card input field (replaces our custom card inputs).
On Pay button click: `stripe.confirmCardPayment(client_secret, { payment_method: { card } })`.
On Stripe success: call `POST /orders/confirm` with `payment_intent_id`.
On success: show Step 3 confirmation screen with order number.

### F3 — Confirm Order
`POST /api/orders/confirm` (protected).
Accepts: `payment_intent_id`.
Retrieves PaymentIntent from Stripe to verify status is `succeeded`.
Creates order record in DB.
Generates one ticket per quantity: unique `ticket_code` (UUID), QR code PNG (base64).
Returns: `{ order_number, tickets: [{ ticket_code, qr_code }] }`.

### F4 — Stripe Webhook
`POST /api/webhooks/stripe` — public endpoint, no auth middleware.
Verifies Stripe signature with `STRIPE_WEBHOOK_SECRET`.
Handles events:
- `payment_intent.succeeded` → mark order as `paid` if not already done
- `payment_intent.payment_failed` → mark order as `failed`, log reason
- `charge.refunded` → mark order as `refunded`
Returns `{ received: true }` immediately (Stripe expects fast response).

### F5 — Get My Orders
`GET /api/orders/me` (protected).
Returns all orders for the authenticated user, newest first.
Includes associated tickets for each order.

### F6 — QR Code Generation
Each ticket gets a unique `ticket_code` (UUID).
QR code encodes: `{ ticket_code, event: 'APEX2026', plan: plan_key }`.
Generated as base64 PNG using `qrcode` npm package.
Stored reference in tickets table. Actual QR sent in email (Phase 4).

## Payment Flow Architecture

```
Browser                     Our Server              Stripe
──────                     ──────────              ──────
[Step 1: Personal Info]
[Step 2: Card]
  ↓ POST /orders/create ──────────────────→ createPaymentIntent()
                         ←────────────────── { client_secret }
  stripe.confirmCardPayment(client_secret)
                         ──────────────────→ Stripe processes card
                         ←────────────────── PaymentIntent: succeeded
  ↓ POST /orders/confirm ──────────────────→ retrieve PaymentIntent
                                             verify status=succeeded
                                             INSERT order + tickets
                         ←────────────────── { order_number, tickets }
  [Step 3: Success ✓]

                         ←────────────────── Webhook: payment_intent.succeeded
                                             (backup — mark order paid)
```

## Stripe Configuration

```js
// backend/services/stripe.js
const Stripe = require('stripe');
const stripe = Stripe(process.env.STRIPE_SECRET_KEY);

// Always pass idempotency keys to avoid duplicate charges
// Use payment_intent_id as idempotency key for /confirm
```

## Amount Calculation

```
base = plan.price * quantity              (e.g. 259 * 2 = 518)
discount = base * promoDiscount           (e.g. 518 * 0.20 = 103.60)
subtotal = base - discount                (e.g. 518 - 103.60 = 414.40)
fee = subtotal * 0.03                    (e.g. 414.40 * 0.03 = 12.43)
total = subtotal + fee                   (e.g. 414.40 + 12.43 = 426.83)
stripe_amount = Math.round(total * 100)  (e.g. 42683 cents)
```

## Order Number Generation

```js
// Format: ALS-XXXXXX (6 random digits)
const orderNumber = 'ALS-' + Math.floor(100000 + Math.random() * 900000);
// Check uniqueness before insert — retry if collision
```

## Ticket Code + QR

```js
const { v4: uuidv4 } = require('uuid');
const QRCode = require('qrcode');

const ticketCode = uuidv4();
const qrData = JSON.stringify({ code: ticketCode, event: 'APEX2026', plan: planKey });
const qrBase64 = await QRCode.toDataURL(qrData);
```

## Webhook Security

```js
const event = stripe.webhooks.constructEvent(
  req.rawBody,          // must use express.raw() for this route, NOT express.json()
  req.headers['stripe-signature'],
  process.env.STRIPE_WEBHOOK_SECRET
);
```

⚠️ The `/webhooks/stripe` route must use `express.raw({ type: 'application/json' })` instead of `express.json()`. Mount it BEFORE the json middleware.

- `generateOrderNumber()` → unique 'ALS-XXXXXX' string (checks DB for collision)

**Verify**: createTickets with quantity=3 creates 3 ticket rows with unique codes.

## Task 3.3 — Build POST /orders/create endpoint

**File**: `backend/routes/orders.js`

Apply `authMiddleware`.
Validate: `plan_key` exists in `plans` table and `is_active = true`, `quantity` is integer 1-20.
If `promo_code` provided: validate via promo_codes table (reuse logic from Phase 5 — stub for now, accept any code with 0 discount).
Call `calculateOrder()`.
Call `stripe.createPaymentIntent(stripeAmount, 'usd', { user_id, plan_key, quantity })`.
Return `{ success: true, data: { client_secret, order_preview } }`.

**Verify**: POST with valid plan_key returns a Stripe client_secret string.

## Task 3.4 — Build POST /orders/confirm endpoint

**File**: `backend/routes/orders.js`

Apply `authMiddleware`.
Accept `payment_intent_id`.
Call `stripe.retrievePaymentIntent(id)`.
If status !== 'succeeded' → return 400 'Payment not completed'.
Check order doesn't already exist for this payment_intent_id (idempotency).
Call `createOrderRecord()` and `createTickets()`.
Return `{ success: true, data: { order_number, tickets } }`.

**Verify**: Calling /confirm twice with same payment_intent_id doesn't create duplicate order.

---

## Task 3.5 — Build Stripe webhook endpoint

**File**: `backend/routes/webhooks.js`

Mount BEFORE `express.json()` in server.js using `express.raw({ type: 'application/json' })`.
Call `stripe.constructWebhookEvent()` — if throws → 400.
Handle `payment_intent.succeeded`: find order by stripe_payment_id, update status to 'paid'.
Handle `payment_intent.payment_failed`: update status to 'failed', log failure reason.
Handle `charge.refunded`: update status to 'refunded'.
Always return `{ received: true }` with 200.

**Verify**: Using Stripe CLI `stripe listen --forward-to localhost:5000/api/webhooks/stripe` — test event delivery succeeds.

---

## Task 3.6 — Build GET /orders/me endpoint

**File**: `backend/routes/orders.js`

Apply `authMiddleware`.
`SELECT orders.*, tickets.* FROM orders LEFT JOIN tickets ON tickets.order_id = orders.id WHERE orders.user_id = $1 ORDER BY orders.created_at DESC`.
Group tickets under each order in response.
Return `{ success: true, data: { orders: [...] } }`.

**Verify**: Returns empty array for new user, populated array after purchase.
