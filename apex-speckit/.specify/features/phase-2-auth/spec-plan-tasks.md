# Phase 2 — Spec: User Authentication
**Spec Kit artifact** · Run `/speckit.specify` then `/speckit.clarify`

---

## Overview

Users can create accounts, sign in, and stay signed in across sessions.
Passwords are hashed. Sessions use JWT in httpOnly cookies.
The frontend auth modal connects to real API endpoints.

---

## Functional Requirements

### F1 — Registration
`POST /api/auth/register` accepts: `first_name`, `last_name`, `email`, `password`, `phone` (optional), `organization` (optional), `country`, `city` (optional).
Validates email format and uniqueness. Password minimum 8 characters.
Hashes password with bcrypt (12 rounds). Inserts user. Returns JWT cookie + user object (no password field).
Sends welcome email via SendGrid.

### F2 — Login
`POST /api/auth/login` accepts `email`, `password`.
Finds user by email. Compares bcrypt hash. If mismatch → 401 with generic message (never specify which field is wrong).
On success: sets `token` httpOnly cookie with JWT. Returns user object.

### F3 — Logout
`POST /api/auth/logout` clears the `token` cookie. Returns `{ success: true }`.

### F4 — Get Current User
`GET /api/auth/me` — protected route.
Returns the authenticated user's profile (no password).

### F5 — Forgot Password
`POST /api/auth/forgot-password` accepts `email`.
Always returns 200 (prevents email enumeration). If email exists: generates reset token, saves hashed version + expiry to user record, sends password reset email with link.

### F6 — Reset Password
`POST /api/auth/reset-password` accepts `token`, `password`.
Validates token exists, matches hash, is not expired (1 hour TTL).
Hashes new password. Clears reset token fields. Returns 200.

### F7 — Auth Middleware
`middleware/auth.js` reads JWT from `req.cookies.token`.
Verifies with `JWT_SECRET`. Attaches `req.user = { id, email, role }`.
On failure → 401. Used by all protected routes.

### F8 — Frontend Integration
Sign In tab in auth modal → `POST /api/auth/login` → store user in React Context → update navbar.
Register tab → `POST /api/auth/register` → same flow.
Navbar shows user first name + avatar initials after login.
Sign In button replaced by user menu with Dashboard + Logout.

---

## Acceptance Criteria
- [ ] Register with valid data → user in DB, cookie set, welcome email sent
- [ ] Register with duplicate email → 409 conflict response
- [ ] Login with correct credentials → cookie set, user returned
- [ ] Login with wrong password → 401, same message as wrong email
- [ ] `GET /api/auth/me` without cookie → 401
- [ ] `GET /api/auth/me` with valid cookie → user object returned
- [ ] Logout clears cookie, subsequent /me returns 401
- [ ] Forgot password with unknown email → still returns 200
- [ ] Reset password with expired token → 400 error

# Phase 2 — Technical Plan: User Authentication
**Generated by `/speckit.plan`**

---

## Architecture

```
routes/auth.js
  ├── POST /register  → validateRegistration[] → registerController
  ├── POST /login     → validateLogin[] → loginController
  ├── POST /logout    → logoutController
  ├── GET  /me        → authMiddleware → getMeController
  ├── POST /forgot-password → forgotPasswordController
  └── POST /reset-password  → resetPasswordController

middleware/auth.js    → verifyJWT → attach req.user
middleware/roleCheck.js → check req.user.role against allowed roles

services/email.js     → sendWelcomeEmail(), sendResetEmail()
```

## JWT Strategy

```
Secret: JWT_SECRET env variable (min 32 chars)
Algorithm: HS256 (default)
Payload: { id, email, role }
Expiry: JWT_EXPIRES_IN (e.g. "7d")
Storage: httpOnly cookie named "token"
Cookie options: httpOnly: true, secure: NODE_ENV==='production', sameSite: 'strict', maxAge: 7*24*60*60*1000
```

## Password Reset Flow

```
1. Generate random token with crypto.randomBytes(32).toString('hex')
2. Hash it with crypto.createHash('sha256')
3. Store hash + expiry (Date.now() + 3600000) in users table
4. Send plain token in email link: /reset-password?token=PLAIN_TOKEN
5. On reset: hash incoming token, compare to DB hash, check expiry
```

## New Columns for Password Reset (users table migration)

```sql
ALTER TABLE users ADD COLUMN IF NOT EXISTS reset_token VARCHAR(255);
ALTER TABLE users ADD COLUMN IF NOT EXISTS reset_token_expires TIMESTAMP;
```

## Rate Limiting — Auth Routes

```js
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,  // 15 minutes
  max: 10,
  message: { success: false, error: 'Too many attempts, try again later' }
});
// Apply to: /register, /login, /forgot-password
```

## Task 2.3 — Build auth middleware

**File**: `backend/middleware/auth.js`

Read `req.cookies.token`. If missing → `res.status(401).json(...)`.
`jwt.verify(token, process.env.JWT_SECRET)`.
On success: `req.user = { id, email, role }` → `next()`.
On failure (expired, invalid): `res.status(401).json(...)`.

**Verify**: Protected route returns 401 without cookie, proceeds with valid cookie.

## Task 2.5 — Build register endpoint

**File**: `backend/routes/auth.js`

`POST /api/auth/register`:
1. Run `validationResult(req)` — if errors → 400 with details
2. Check if email exists in DB → 409 if yes
3. `bcrypt.hash(password, 12)`
4. `INSERT INTO users ...` returning all fields except password
5. Sign JWT: `jwt.sign({ id, email, role }, JWT_SECRET, { expiresIn })`
6. `res.cookie('token', jwt, cookieOptions)`
7. Call `sendWelcomeEmail(user)` (non-blocking — don't await)
8. Return `{ success: true, data: { user } }`

**Verify**: POST with valid body → 201, cookie in response headers, user in DB.

---

## Task 2.6 — Build login endpoint

**File**: `backend/routes/auth.js`

`POST /api/auth/login`:
1. Validate fields
2. `SELECT * FROM users WHERE email = $1` — if not found → 401 with GENERIC message
3. `bcrypt.compare(password, user.password)` — if false → 401 with SAME generic message
4. Sign JWT, set cookie
5. Return user object (exclude password field from response)

**Verify**: Wrong email and wrong password return identical 401 response body.

---

## Task 2.7 — Build logout + me endpoints

**File**: `backend/routes/auth.js`

`POST /api/auth/logout`: `res.clearCookie('token')` → `{ success: true }`.
`GET /api/auth/me`: apply `authMiddleware` → `SELECT user WHERE id = req.user.id` → return user (no password).

**Verify**: Logout then /me returns 401.

---

## Task 2.8 — Build forgot + reset password endpoints

**File**: `backend/routes/auth.js`

`POST /api/auth/forgot-password`:
- Always return 200. Internally: find user, generate token, hash+store, send email.

`POST /api/auth/reset-password`:
- Hash incoming token, find user where reset_token = hash AND reset_token_expires > NOW().
- If not found → 400 'Invalid or expired token'.
- Hash new password, update user, clear reset_token fields.

**Verify**: Valid token resets password. Expired token (manually set to past) returns 400.
